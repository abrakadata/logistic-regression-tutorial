① The Problem We're Solving

Before we touch any math, let's be crystal clear about what we want the computer to do.

Imagine you're a professor. You've been teaching for years, and you notice a pattern: students who attend class regularly and study a good number of hours tend to pass. Students who skip class and barely study tend to fail. You want to capture that pattern as a formula, so you can predict a new student's outcome.

🎯 Our Goal

Given a student's class attendance % and weekly study hours, predict whether they will PASS (1) or FAIL (0).

Our Training Data

A dataset is just a collection of past examples. Here is our dataset — 8 students whose outcomes we already know. We'll use these examples to "teach" the model.

Notice: the result is always 0 or 1. Nothing in between. This is called a binary classification problem — "binary" because there are exactly two possible outcomes. Logistic Regression is perfectly designed for this type of problem.

Key Terms We Just Introduced

📖 Vocabulary

Feature — An input variable used to make a prediction. We have two features: Attendance and Study Hours.

Label — The answer we're trying to predict. Here, it's Pass (1) or Fail (0).

Training data — Historical examples with known answers that we use to teach the model.

Binary classification — A prediction task with exactly two possible outcomes.

② Linear Math Refresher

To understand Logistic Regression, we first need to revisit two simple ideas from math: weighted sums and the equation of a line. Don't worry — we'll build these up from plain English.

What is a Weighted Sum?

Imagine you're judging a job applicant. You care about two things: their GPA and their years of experience. But you don't care equally about both — you think experience matters twice as much as GPA. So you compute a score like this:

Score formula:Score = (2 × Experience) + (1 × GPA)
Example applicant:Experience = 3 years, GPA = 3.5
Calculation:Score = (2 × 3) + (1 × 3.5) = 6 + 3.5 = 9.5

The numbers 2 and 1 are called weights. They express how much you care about each input. A higher weight = more influence on the final score.

In machine learning, we call these weights parameters or coefficients. The model learns the best weights from data. We also add a special extra number called the bias (sometimes called the intercept), which lets the model shift its output up or down even when all inputs are zero.

Weighted Sum (Linear Combination)
\(z = w_1x_1 + w_2x_2 + b\)
x₁ = Attendance, x₂ = Study Hours
w₁, w₂ = weights (how important each feature is)
b = bias (a baseline shift)
z = the raw score (we'll transform this later)

What Does the Dot (·) Mean?

The dot between w₁ and x₁ just means multiply. So w₁·x₁ = w₁ × x₁. It's shorthand mathematicians use to avoid confusion with the letter "x".

The Line Intuition

You may recall from school: y = mx + b, the equation of a straight line. Our weighted sum formula is exactly the same idea, just extended to two inputs instead of one:

💡 Key Insight

z = w₁·x₁ + w₂·x₂ + b is just a straight line in disguise. With two features, it's actually a flat surface (a plane) in 3D space — but the concept is identical. The weights control the slope; the bias controls the height.

Let's Try a Concrete Example

Say we guess (just randomly for now) these weights: w₁ = 0.05, w₂ = 0.2, b = −5. Let's compute z for Alice (attendance=90%, study hours=8):

Formula:z = w₁·x₁ + w₂·x₂ + b
Plug in:z = 0.05×90 + 0.2×8 + (−5)
Calculate:z = 4.5 + 1.6 − 5
Result:z = 1.1

So Alice gets a raw score of 1.1. But what does 1.1 mean? Is she going to pass? That's the problem with raw scores — they can be anything from −∞ to +∞. We need to convert them into a probability between 0 and 1. That's coming in Section 4.

③ Why Not Linear Regression?

A fair question: we already know how to draw a line through data points — that's called Linear Regression. Why can't we just use that to predict Pass/Fail?

What Linear Regression Does

Linear Regression predicts a continuous number — things like house prices, temperature, salary. It fits a straight line to the data and predicts values anywhere along that line.

If we tried to apply Linear Regression to our Pass/Fail problem, we'd run into two serious problems:

Problem 1: Outputs That Don't Make Sense

A straight line has no limits. It can produce outputs like 1.7, 3.4, or −0.5. But probabilities must be between 0 and 1. What does a probability of 1.7 or −0.5 mean? Nothing. It's mathematically invalid.

Problem 2: The Decision Boundary Breaks Down

Suppose a student studies 50 hours per week (an extreme outlier who aces everything). A straight line would be "pulled" toward this extreme point and start predicting that students with moderate attendance have a lower chance of passing — which is backward.

⚠️ The Core Problem

Linear Regression assumes outputs can go to infinity in both directions. Classification problems need outputs bounded between 0 and 1. We need a function that takes any number and squashes it into the [0,1] range.

What We Need Instead

We need a mathematical function that:

✅ Takes any number as input (from −∞ to +∞)

✅ Always outputs a value between 0 and 1

✅ Outputs values near 1 for large positive inputs

✅ Outputs values near 0 for large negative inputs

✅ Outputs exactly 0.5 when the input is 0 (perfectly uncertain)

That magical function exists — and it's called the Sigmoid function. Let's meet it in the next section.

④ The Sigmoid Function

The Sigmoid function (also called the logistic function — which is where Logistic Regression gets its name) is the heart of our model. Its job: take any number and squash it into a probability between 0 and 1.

The Sigmoid Function
\(\sigma(z) = \frac{1}{1 + e^{-z}}\)
σ (sigma) is just the name of this function
z = our weighted sum from before
e ≈ 2.718 (Euler's number, a mathematical constant like π)
Output is always between 0 and 1

What Does e Mean?

The letter e (Euler's number) is a special mathematical constant, approximately 2.718. Just like π ≈ 3.14159 appears naturally in circles, e appears naturally in growth and decay problems. You don't need to memorize it — just know it's a fixed number your calculator knows.

Let's Compute It For Alice

We calculated Alice's raw score z = 1.1. Let's pass it through the Sigmoid:

Formula:σ(z) = 1 / (1 + e^−z)
Plug in z=1.1:σ(1.1) = 1 / (1 + e^−1.1)
Compute e^−1.1:e^−1.1 ≈ 0.333
Add 1:1 + 0.333 = 1.333
Divide:1 / 1.333 ≈ 0.75
Interpretation:Alice has a 75% chance of passing

The Beautiful Properties of Sigmoid

✅ Why Sigmoid Works Perfectly

When z = 0 → σ(0) = 0.5 (50/50 — perfectly uncertain)

When z is very large (+∞) → σ → 1 (almost certain to pass)

When z is very negative (−∞) → σ → 0 (almost certain to fail)

The output is always a valid probability. Problem solved.

⑤ Building the Model

Now we can combine everything into our Logistic Regression model. The model is a two-step process:

1

Compute the Weighted Sum

Take the student's features (attendance, study hours), multiply each by its weight, add the bias. This gives us a raw score z that can be any number.

\(z = w_1\,\mathrm{Attendance} + w_2\,\mathrm{StudyHours} + b\)
2

Pass Through the Sigmoid

Feed z into the Sigmoid function. The output is a number between 0 and 1 — that's the model's predicted probability of passing.

\(P(\mathrm{pass}) = \sigma(z) = \frac{1}{1 + e^{-z}}\)
3

Apply a Decision Threshold

If the probability ≥ 0.5, predict PASS. If < 0.5, predict FAIL. The 0.5 threshold is the most common choice, but it can be adjusted for different situations (e.g., if failing is very costly, you might set the threshold higher).

The Complete Model — Combined Formula

Complete Logistic Regression Model
\(P(\mathrm{pass}) = \frac{1}{1 + e^{-(w_1x_1 + w_2x_2 + b)}}\)
That's it. That's the entire Logistic Regression model.
The only unknowns are w₁, w₂, and b — the parameters the model must learn.

What Does "Learning" Mean?

Right now, we just made up some weights (w₁=0.05, w₂=0.2, b=−5). The model learns by finding the weights that make it most accurate on the training data. But how? That's the job of an algorithm called Gradient Descent, which we'll explore in Section 7.

First, let's make predictions with our current (random) weights and see how the model behaves.

💡 Why This is Called "Logistic" Regression

The name comes from the logistic function, which is another name for the Sigmoid. And even though it's called "regression" (usually associated with predicting numbers), Logistic Regression is actually a classification model. The naming is a historical quirk — the method uses a regression-like formula internally, but produces a classification output.

⑦ How the Model Learns

This is the most important — and most fascinating — part. How does the model actually find the right weights? The answer involves three ideas: a loss function, derivatives, and gradient descent. Let's build each one up from scratch.

Step 1: Measuring How Wrong We Are — The Loss Function

Before we can improve, we need to measure how bad our current weights are. This measurement is called the loss (or cost). The lower the loss, the better the model.

For Logistic Regression, we use a special loss called Binary Cross-Entropy Loss. Don't let the name intimidate you — the intuition is simple:

🎯 Intuition

If the true answer is PASS (1) and our model predicted 0.95 probability → small penalty (close!)

If the true answer is PASS (1) and our model predicted 0.05 probability → huge penalty (very wrong!)

The loss function captures this: it punishes confident wrong predictions much more than uncertain wrong predictions.

Binary Cross-Entropy Loss (for one student)
\(L = -\left[y\log(p) + (1-y)\log(1-p)\right]\)
y = true label (0 or 1)
p = model's predicted probability
log = natural logarithm (a math function your calculator knows)
When y=1: L = −log(p) — rewards high p, punishes low p
When y=0: L = −log(1−p) — rewards low p, punishes high p

The total loss is the average of this over all students. Our goal: find weights that minimize the total loss.

Step 2: Derivatives — Which Direction to Improve?

A derivative tells you: if I change this value a tiny bit, how much does the output change? Think of it as the slope at a specific point on a curve.

⛰️ Mountain Analogy

Imagine you're blindfolded on a hilly landscape. Your goal is to reach the lowest valley (minimum loss). You can feel the slope under your feet. If the ground slopes up to your right, you should step left. If it slopes up to your left, you step right. The derivative tells you the slope — so you always know which way is "downhill".

This is called Gradient Descent — descending along the gradient (slope) of the loss.

Step 3: Gradient Descent — The Learning Algorithm

Gradient Descent works by repeatedly adjusting the weights in the direction that decreases the loss. Here's the update rule:

Gradient Descent Update Rule
\(w_1^{(new)} = w_1 - \alpha \frac{\partial L}{\partial w_1}\)
w₁(new) means the updated value of w₁ after one step
α (alpha) = learning rate — how big a step we take (e.g. 0.01)
∂L/∂w₁ = partial derivative of Loss with respect to w₁
"partial" just means: how does L change if we nudge w₁, holding everything else fixed?

Same update applies to w₂ and b.

The Actual Derivatives (Simplified)

For Logistic Regression, the math works out to an elegant formula. The gradient for each weight is just the average error multiplied by the corresponding feature:

Gradients for Logistic Regression
\(\frac{\partial L}{\partial w_1} = \frac{1}{N} \sum (p - y) x_1\)
N = number of students (8 in our case)
Σ (sigma) = "sum over all students"
(p − y) = prediction minus true label (the "error")
x₁ = the feature value (attendance) for that student

Same formula for w₂ (replace x₁ with x₂) and b (no feature multiplication)

Why Does This Formula Make Sense?

💡 Intuition Behind the Gradient

When p − y = 0, we made a perfect prediction — no adjustment needed.

When p − y > 0 (we predicted too high), the gradient is positive, so we subtract from the weight (reduce it).

When p − y < 0 (we predicted too low), the gradient is negative, so we add to the weight (increase it). The feature x₁ acts as a scale — features with larger values get larger gradient nudges.

One Complete Training Step

Let's manually do one gradient descent step with α = 0.01 and our random initial weights (w₁=0.05, w₂=0.2, b=−5), computing the gradient using all 8 students. The math here is just arithmetic — the same formula repeated 8 times and averaged:

Compute predictions:Run all 8 students through current model
Compute errors:p − y for each student
Average gradient w₁:≈ −0.142 (w₁ needs to increase)
Average gradient w₂:≈ −0.081 (w₂ needs to increase)
Average gradient b:≈ −0.213 (b needs to increase)
Update w₁:0.05 − 0.01×(−0.142) = 0.0514
Update w₂:0.2 − 0.01×(−0.081) = 0.2008
Update b:−5 − 0.01×(−0.213) = −4.9979
Repeat 1000s of times...Until loss is minimized

What You're Seeing

Each point on the loss curve is one epoch. As gradient descent runs:

✅ The loss curve goes downward — the model is getting better

✅ The weights change to better capture the pattern in the data

✅ Eventually the curve flattens out — convergence! The model has found the best weights it can.

📖 New Term: Epoch

An epoch is one complete pass through the entire training dataset. If we run 30000 epochs, the model looks at all 8 students 30000 times, each time updating its weights slightly. More epochs generally means better learning, up to a point.

What Happens If the Learning Rate (α) Is Wrong?

The learning rate is one of the most important settings (called a hyperparameter — a setting you choose before training, unlike weights which are learned):

🐌 Too small (e.g., 0.0001): Training takes forever — tiny steps toward the minimum

🏃 Just right (e.g., 0.01): Smooth, efficient descent to the minimum

💥 Too large (e.g., 10): The model "overshoots" and bounces around, loss goes up instead of down

⑨ The Full Picture

You now understand every piece of Logistic Regression. Let's put it all together, and then look at a clean Python implementation.

The Complete Process

1

Collect Training Data

Gather historical examples with features (inputs) and labels (known outcomes).

2

Initialize Weights

Start with random (or zero) weights w₁, w₂, and bias b.

3

Forward Pass

Compute z = w₁x₁ + w₂x₂ + b for every student, then p = σ(z).

4

Compute Loss

Calculate the average Binary Cross-Entropy Loss across all training examples.

5

Backward Pass (Gradients)

Compute how much each weight contributed to the loss using partial derivatives.

6

Update Weights

Adjust each weight by subtracting learning_rate × gradient.

7

Repeat

Go back to step 3. Repeat for many epochs until the loss converges.

8

Predict New Students

For a new student, compute z → σ(z) → if ≥ 0.5, predict PASS.

Python Implementation From Scratch

Here's the entire model in clean Python — no external libraries, no magic. Just the math we walked through:

# Logistic Regression From Scratch
# Predicting student pass/fail

import math

# ── Our training data ──────────────────────
# [attendance%, study_hours]
X = [
[90, 8], [30, 2], [75, 6], [50, 3],
[85, 7], [20, 1], [65, 5], [40, 2]
]
y = [1, 0, 1, 0, 1, 0, 1, 0] # 1=pass, 0=fail

# ── Sigmoid function ───────────────────────
def sigmoid(z):
return 1 / (1 + math.exp(-z))

# ── Initialize weights randomly ────────────
w1 = 0.0
w2 = 0.0
b = 0.0
lr = 0.01 # learning rate
epochs = 1000

# ── Training loop ──────────────────────────
for epoch in range(epochs):

# Step 1: Forward pass — compute predictions
predictions = []
for student in X:
z = w1 * student[0] + w2 * student[1] + b
p = sigmoid(z)
predictions.append(p)

# Step 2: Compute loss (Binary Cross-Entropy)
loss = 0
for i in range(len(y)):
p = predictions[i]
loss -= y[i] * math.log(p + 1e-9)
loss -= (1 - y[i]) * math.log(1 - p + 1e-9)
loss /= len(y)

# Step 3: Compute gradients
grad_w1 = 0
grad_w2 = 0
grad_b = 0
N = len(y)
for i in range(N):
error = predictions[i] - y[i]
grad_w1 += error * X[i][0]
grad_w2 += error * X[i][1]
grad_b += error
grad_w1 /= N
grad_w2 /= N
grad_b /= N

# Step 4: Update weights
w1 -= lr * grad_w1
w2 -= lr * grad_w2
b -= lr * grad_b

if epoch % 100 == 0:
print(f"Epoch {epoch}: Loss = {loss:.4f}")

# ── Prediction function ────────────────────
def predict(attendance, study_hours):
z = w1 * attendance + w2 * study_hours + b
probability = sigmoid(z)
label = "PASS" if probability >= 0.5 else "FAIL"
print(f"P(pass) = {probability:.1%} → {label}")

# ── Try it out ─────────────────────────────
predict(80, 7) # Should be PASS
predict(25, 2) # Should be FAIL
predict(55, 4) # Borderline case
